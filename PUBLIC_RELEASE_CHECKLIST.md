# PUBLIC_RELEASE_CHECKLIST

Use this before publishing FlowMotion Skill.

## A. Privacy

- [ ] No local absolute paths.
- [ ] No private transcripts.
- [ ] No private chat logs.
- [ ] No internal score notes.
- [ ] No private project names.
- [ ] No API keys, tokens, cookies, emails, or phone numbers.
- [ ] No hidden local browser or rendering scripts.

## B. Third-Party Material

- [ ] No third-party screenshots.
- [ ] No third-party recordings.
- [ ] No copyrighted source text.
- [ ] No copied external code.
- [ ] Examples use synthetic or public-safe text.

## C. Skill Naming

- [ ] Repository or folder may be `flowmotion-skill-public`.
- [ ] Skill display name remains `FlowMotion Skill`.
- [ ] `skill/SKILL.md` frontmatter name does not include `public`.
- [ ] README explains public release without renaming the Skill.

## D. Function Boundary

- [ ] Public version opens the method, not the full private workflow.
- [ ] No private multi-skill chain is required to understand the public package.
- [ ] No claim that the public package reproduces all social media demos.
- [ ] Motion examples are briefs/templates, not private production files.

## E. Release Readiness

- [ ] README has first-screen hook.
- [ ] README has install instructions.
- [ ] README has examples and templates.
- [ ] LICENSE exists.
- [ ] BRAND_NOTICE exists.
- [ ] GITHUB_ABOUT_SUGGESTION exists.
